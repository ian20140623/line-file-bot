## Phase B：分流處理
- [ ] B1. 圖片分流
  - [ ] 收到圖片 → 自動 OCR 預覽 → 問用戶：「1. 報告審稿  2. 文字提取」
  - [ ] 文字提取模式：問「全部提取」or「選擇段落」
  - [ ] 段落選擇：顯示段落預覽（日期、第一段開頭、第二段開頭…），用戶點選要哪些
  - [ ] 報告審稿模式：
    1. 提取全文
    2. 校對：錯字、錯誤標點、異常空白
    3. 事實查核：辨識文中引用的新聞事件（尤其數字）
       - 列出偵測到的新聞事件，讓用戶勾選要確認哪些
       - 選定的事件：搜尋對照新聞，回傳連結供比對
- [ ] B2. PDF 智慧分流
  - [ ] 收到 PDF（通常一批）→ 自動判斷是否為券商報告
  - [ ] **是券商報告** → 同一批一起問用戶（可能用時間框架）：
    1. 快速摘要
    2. 詳細摘要
    3. 閱讀內文的文字擷取版（去除 ESG — broker-reports 已做相關功能）
    4. 存檔至預設雲端硬碟 + 去重 + 走 broker-reports 流程提取 MD
  - [ ] **非券商報告** → 問用戶：「1. 報告審核  2. 文字提取」
  - [ ] 去重機制：content hash 比對已處理檔案
- [ ] B3. 行程/會議解析
  - [ ] 偵測行程/會議相關內容（文字、圖片、照片皆可）
  - [ ] 解析關鍵資訊：時間、人物、地點、議題
  - [ ] 產出 .ics 行事曆檔
  - [ ] 券商資訊後綴：行程標題自動加上「-元大」「-永豐」等
  - [ ] 人名對照表：券商 ↔ 營業員對照（JSON）
- [ ] B4. 多檔案自動關聯
  - [ ] 2 小時內多檔案自動判斷是否同一場活動
  - [ ] 手動關聯：超出時間窗口的檔案可手動指定
  - [ ] 合併處理：交叉整合產出完整紀錄
